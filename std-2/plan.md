# 国家药品带量采购AI分析系统 - 技术方案

## 一、项目背景

### 目标
构建一个AI驱动的药品带量采购监管分析系统，通过调用MiniMax大语言模型，对集采执行进度进行智能监管并提供专业建议。

### 场景
- 集采执行进度监控
- 异常数据预警
- 合规性审查建议
- 采购数据分析

## 四、28原则核心逻辑

- **定义**：20%的医院合同量大但采购量小，导致整体采购进度滞后
- **影响**：这20%关键医院的执行情况直接决定全省/地市的整体进度
- **重点监管**：找出这些"合同量大、采购量小"的医院，作为重点监管对象

```
关键医院（合同量大、采购量小）
    │
    ├── 采购进度快 → 整体进度正常
    │
    └── 采购进度慢 → 导致地市/全省进度滞后 ⚠️ 重点监管对象
```

### 状态 (State) 定义

```python
class AnalysisState(TypedDict):

    region_analysis: str              # 地市医院分析结果

```

## 八、序时进度指标

### 计算公式
```
序时进度 = (合同总量 / 365 × (当前时间 - 批次执行时间)) / 合同总量
简化后：序时进度 = (当前时间 - 批次执行时间) / 365
```

### 进度判定
| 状态 | 条件 | 说明 |
|------|------|------|
| 超序时 | 实际进度 > 序时进度 | 进度超前 |
| 正常 | 实际进度 ≈ 序时进度 | 进度正常（±10%容差）|
| 落后 | 实际进度 < 序时进度 | 进度滞后，需要预警 |

| `/api/analyze` | GET | AI流式分析（SSE）- 报告模式 |
| `/api/chat` | GET | AI流式对话（SSE）- 问答模式 |
| `/api/health` | GET | 健康检查 |

### 两种对话模式

1. **报告模式** (`/api/analyze`)：点击分析按钮，生成完整监管报告
2. **问答模式** (`/api/chat`)：用户自由提问，返回流式回答

### MiniMax API流式调用

```python
# 流式输出核心实现
from flask import Response, stream_with_context

@app.route('/api/analyze')
def analyze():
    def generate():
        client = MiniMax(api_key="your-key", group_id="your-group")
        stream = client.chat.completions.create(
            model="abab6.5s-chat",
            messages=[{"role": "user", "content": prompt}],
            stream=True
        )
        for chunk in stream:
            if chunk.choices[0].delta.content:
                yield f"data: {chunk.choices[0].delta.content}\n\n"

    return Response(
        stream_with_context(generate()),
        mimetype='text/event-stream',
        headers={'Cache-Control': 'no-cache'}
    )
```
